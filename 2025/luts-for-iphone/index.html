最近富士公开了其经典的十数种 LUT 滤镜，包括著名的 CC 滤镜和 NC 滤镜，使得其他拍摄设备，也能最大限度还原得到富士相机的效果，而不需要购入对应的富士相机。但在将富士LUTs应用到iPhone前，不少人是第一次接触到这一概念，这需要我们花点儿时间，来理解背后的色彩科学。

笔者通过编程方式，结合实际数据和图表，力图深入浅出解释LUT、Log模式以及为什么需要进行色彩空间转换。

1. 什么是 LUT (Look-Up Table)

LUT 本质上是一个映射表，就如其英文全称所言。对于 3D LUT，它是一个 RGB 立方体。输入一个 RGB 值 (R, G, B)，查找表，输出一个新的 RGB 值。

LUT 的核心映射逻辑

  1. 输入 RGB: 假设视频里有一个像素，颜色是 R=0.5, G=0.5, B=0.5 (中性灰)。
  2. 查找 (Look Up): LUT 文件是一个三维数组 (例如 33x33x33)。
    • 我们把 0.0~1.0 的范围划分成 33 个格子。
    • 0.5 刚好对应第 17 个格子索引 (因为 16/32 = 0.5)。
  3. 定位: 找到坐标为 (17, 17, 17) 的那个小格子。
  4. 读取: 读取这个格子预先存储的 RGB 值。比如，胶片模拟可能会把这个灰变成略带冷调的 (0.48, 0.49, 0.52)。
  5. 插值 (Interpolation): 如果输入是 0.51，正好在两个格子之间，计算机会根据周围的格子计算出一个中间值 (三线性插值)。

2. 富士 LUT 的规格：Grid 33 vs Grid 65

富士官方提供了两种精度的 LUT。这个数字 (33 或 65) 代表了 RGB 立方体的边采样点数（上图为Grid 5）

  • 33 Grid: $33^3 = 35,937$ 个数据点。文件较小 (~1MB)，计算快，适合大多数实时渲染场景。
  • 65 Grid: $65^3 = 274,625$ 个数据点。文件较大 (~7MB)，精度高，适合后期调色 (如 DaVinci Resolve)。
  • 数据量对比: 65 Grid 的数据点数量是 33 Grid 的 7.64 倍。

3. 深入解析：Log 模式与灰度之谜

什么是 Log 模式，与普通拍照有何不同？

Log (Logarithmic, 对数) 模式是一种特殊的伽马 (Gamma) 曲线编码方式。它是为了存储更多信息而存在。

  • 普通模式 (Standard/Linear/Rec.709):
    • 映射逻辑：“所见即所得”
    • 曲线特点：S 型曲线。暗部被压暗，亮部被提亮，对比度高
    • 缺点：为了高对比度，舍弃了极亮和极暗的信息（过曝死白，欠曝死黑）
  • Log 模式 (F-Log/Apple Log):
    • 映射逻辑：“压缩映射”
    • 曲线特点：对数曲线。它把真实世界中非常宽广的亮度范围（比如 10000 尼特的太阳到 0.01 尼特的阴影），通过数学公式“压缩”进有限的文件数值里 (0.0 - 1.0)。

为什么 Log 看起来是灰的？

  1. 对比度 = 斜率 (Slope): 在图像中，对比度取决于亮度曲线的斜率。斜率越陡，对比度越高。
  2. 压缩后果: 为了塞进更多的高光细节（如保留天空的云彩），Log 曲线必须在中间调保持非常平缓的斜率。
  3. 平缓斜率 = 低对比度: 既然中间调的斜率平缓，那么原本差异很大的明暗，在 Log 文件里数值差异就很小。人眼看起来就是灰蒙蒙的。

为什么 LUT 要搭配 Log 使用？

LUT (Look-Up Table) 在这里扮演了解码器的角色。Log 负责把更多的信息压缩进小文件，LUT 负责把小文件还原为人眼习惯的对比度和色彩，并顺便加上特定风格。

4. 为什么要转换？(Apple Log vs F-Log2)

在适当配置后，iPhone 拍摄的是 Apple Log。而富士的 LUT 期望输入是 F-Log 或 F-Log2。

如果直接把富士 LUT 套在 Apple Log 素材上，由于 Gamma 曲线不匹配，会导致画面反差错误（通常是高光过曝或暗部死黑），色彩也会发生偏移。

由于富士给出的更全面的LUTs是在F-Log2基础上配置的，所以下图仅对比 Apple Log 与 F-Log2。实际上 Apple Log 与 F-Log 差别较小。

转换逻辑

为了正确使用 LUT，我们需要如下步骤：

  1. Input: Apple Log (iPhone 拍摄源文件)
  2. Step 1: Apple Log -> Linear (还原为线性光)
  3. Step 2: Linear -> F-Log2 (模拟富士相机的 Log 信号)
  4. Step 3: Apply Fuji LUT (应用官方胶片模拟)
  5. Output: Rec.709 (标准屏幕显示)

通过反解，即可得到我们所需的适配 Apple Log 的 LUT。

5. 不同胶片的用途指南

富士提供了多种胶片模拟，每种都有其独特的适用场景：

  1. PROVIA (Standard): 标准模式。适合各种场景，万金油选择。
  2. Velvia (Vivid): 高饱和，高对比。适合风景、自然，不适合人像。
  3. ASTIA (Soft): 柔和。适合时尚人像、静物。在保持鲜艳的同时，对肤色有特殊保护。
  4. Classic Chrome: 纪实风格。低饱和，强对比，略微偏冷。适合街拍、纪实、人文。这是最受欢迎的富士模拟之一。
  5. PRO Neg. Hi/Std: 专业负片。适合影棚人像。Std 版本非常平滑，适合后期调色。
  6. ETERNA: 电影感。超低饱和，超低对比，高动态范围。适合视频创作，非常有电影质感，但直出可能显得太灰。
  7. Acros: 黑白。相比普通黑白，它对阴影细节和颗粒感有特殊处理。
  8. Classic Neg. (Superia): 也是大家常说的胶片。高对比，独特的色彩偏移（红色偏橘，绿色偏青）。适合日常随拍。

6. 我该如何应用LUT

目前，根据苹果官网介绍：要拍摄 Log 格式视频，需要使用运行 iOS 15.1 或更高版本的 13 Pro 及以后的机型，并在相机设置中将 ProRes 编码设置为 Log。为了更好体验，建议使用 iPhone15 Pro 及以上 Pro 机型。

另外，目前苹果本身的相机不支持导入LUT，建议下载免费软件 BlackmagicCam，进行导入。需要注意的是，录制时色彩空间应当设置为 Apple Log。
